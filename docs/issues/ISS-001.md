# unused-file & dead-export : entry points framework manquants

**Règles** : `unused-file`, `dead-export`
**Impact** : ~268 faux positifs sur Amadeo (148 + 120), ~53% du bruit total
**Priorité** : critique

## Problème

Le graph de reachability (`ImportIndex::reachable_from`) et le filtre `dead-export` s'appuient sur les `FrameworkDef` TOML pour identifier les entry points et magic exports. Mais plusieurs conventions framework ne sont pas couvertes :

### 1. TanStack Router — file-based routing

Les fichiers `*.lazy.tsx` et les fichiers de route générés par TanStack Router ne sont pas déclarés comme entry points. Aucun TOML `tanstack-router.toml` n'existe.

**Conventions à ajouter :**
- `detection.dependencies` : `["@tanstack/react-router", "@tanstack/start"]`
- `entry_points.dirs` : `["/routes/"]`
- `entry_points.files` : fichiers matchant `*.lazy.tsx`, `__root.tsx`, `routeTree.gen.ts`
- `magic_exports` : `["Route", "default", "loader", "beforeLoad", "component", "pendingComponent", "errorComponent"]`

### 2. shadcn/ui — bibliothèque de composants locale

Les composants shadcn/ui vivent dans `src/components/ui/` et sont une bibliothèque interne — ils ne sont pas tous importés à tout moment mais ne sont pas "dead" pour autant.

**Options :**
- Ajouter un mécanisme `entry_points.dirs` pour `["/components/ui/"]` via un TOML `shadcn.toml` (détection : présence de `components.json` à la racine)
- Ou : traiter tout dossier contenant un barrel `index.ts` qui ré-exporte comme un "library dir"

### 3. Fichiers de config racine

`dead-export` flagge `export default defineConfig(...)` dans `vite.config.ts`, `playwright.config.ts`, etc. La fonction `is_entry_point` dans `dead_export/text.rs:91-109` ne couvre que les `main.*` et `index.*` racine — pas les fichiers de config.

`unused-file` les exclut déjà via `is_config_file` (`unused_file/text.rs:122-134`), mais `dead-export` n'a pas le même filtre.

**Fix :** appliquer le même `is_config_file` check dans `dead_export/text.rs`.

### 4. Templates email

Les fichiers de template email (ex: `change-email.tsx`) ne sont importés par aucun code frontend — ils sont rendus côté serveur. Ils apparaissent comme orphelins.

**Fix possible :** ajouter un pattern d'entry point pour les dossiers `emails/` ou `templates/` dans un TOML générique, ou permettre un `comply.toml` projet avec `entry_points.dirs` custom.

## Fichiers concernés

- `src/rules/unused_file/text.rs` — logique BFS et `is_entry_point`
- `src/rules/dead_export/text.rs` — logique per-export et `is_entry_point`
- `src/frameworks/mod.rs` — registre TOML, array `RAW`
- `src/frameworks/*.toml` — ajouter `tanstack-router.toml`, `shadcn.toml`
- `src/project/mod.rs:454-482` — helpers `framework_*`
