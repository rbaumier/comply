# Règles techno-spécifiques qui se déclenchent sans vérifier le framework

**Règles** : `rn-no-inline-styles`, `import-dynamic-import-chunkname`, `playwright-no-hooks`, `no-done-callback`
**Impact** : ~38 faux positifs sur Amadeo
**Priorité** : haute
**Statut** : partiellement résolu (merge worktree-fp-hunting) — playwright-no-hooks gatée sur @playwright/test import ; rn-no-inline-styles, import-dynamic-import-chunkname, no-done-callback toujours ouverts

## Problème

Ces règles ciblent une techno précise mais ne consultent pas `ProjectCtx::detected_frameworks` avant de se déclencher. Elles se fient uniquement au langage du fichier (TS_FAMILY) et à des heuristiques de nom de fichier.

### `rn-no-inline-styles` (7 FP)

Règle React Native appliquée sur un projet web Vite + TanStack. Les inline styles flaggés sont dans des templates email où c'est la seule option (pas de CSS en email HTML).

**Fix :** ne déclencher que si `react-native` est dans les dépendances détectées. Ajouter un `rn.toml` (ou `react-native.toml`) au registre framework si absent.

### `import-dynamic-import-chunkname` (12 FP)

Exige `/* webpackChunkName */` dans les `import()`. Le projet utilise Vite/Rollup — ces commentaires n'ont aucun effet.

**Fix :** ne déclencher que si `webpack` est dans les dépendances détectées. Ou mieux : marquer cette règle comme `disabled = true` par défaut (c'est un vestige Webpack-only).

### `playwright-no-hooks` (9 FP)

Le ciblage fichier (`playwright_no_hooks/typescript.rs:5-10`) utilise des marqueurs génériques (`.test.`, `.spec.`, `__tests__`) qui matchent aussi les fichiers Vitest et Jest. La règle interdit `beforeEach`/`afterEach` qui sont idiomatiques dans ces frameworks.

**Fix :** vérifier que `@playwright/test` est dans les dépendances détectées ET que le fichier est dans un dossier Playwright (e.g. `/e2e/`, `/tests-e2e/`) ou importe depuis `@playwright/test`.

### `no-done-callback` (10 FP)

Même problème — appliqué aux fichiers Vitest et Playwright E2E sans distinction du framework de test.

**Fix :** même approche — conditionner sur le framework de test détecté.

## Pattern commun

Le problème sous-jacent est que les règles à `register_ts_family!` s'appliquent à TOUT fichier TS sans filtre framework. Il faudrait un mécanisme pour qu'une règle déclare ses `required_frameworks` dans `RuleMeta` ou dans `register()`, et que le runtime skip automatiquement les fichiers de projets où le framework n'est pas détecté.

## Fichiers concernés

- `src/rules/rn_no_inline_styles/mod.rs` et `typescript.rs`
- `src/rules/import_dynamic_import_chunkname/mod.rs` et `typescript.rs`
- `src/rules/playwright_no_hooks/mod.rs` et `typescript.rs`
- `src/rules/no_done_callback/mod.rs` et `typescript.rs`
- `src/rules/mod.rs` — `RuleMeta` ou `register()` pour ajouter `required_frameworks`
- `src/frameworks/mod.rs` — vérifier que les TOML couvrent `react-native`, `webpack`
