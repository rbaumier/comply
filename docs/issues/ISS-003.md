# no-side-effects-in-initialization se déclenche sur les fichiers de test

**Règle** : `no-side-effects-in-initialization`
**Impact** : 22 faux positifs sur Amadeo
**Priorité** : haute

## Problème

La règle flagge les appels top-level comme `test.describe()`, `test()`, `describe()`, `it()` dans les fichiers de test. Ces appels sont le pattern standard de Playwright, Vitest, et Jest — les fichiers de test ne sont jamais bundlés pour la production, donc les préoccupations de tree-shaking ne s'appliquent pas.

Le code actuel (`no_side_effects_in_initialization/typescript.rs:51-79`) ne fait aucune vérification de type de fichier — il se déclenche sur tout fichier TS-family dont le top-level contient un `call_expression` ou `new_expression`.

## Fix proposé

Ajouter un early return si le fichier est un fichier de test. La logique existe déjà dans `unused_file/text.rs:136-141` :

```rust
const TEST_MARKERS: &[&str] = &[".test.", ".spec.", "__tests__", "_test.", ".e2e."];
```

Alternativement, utiliser `ctx.file.path_segments.in_test_dir` si cette propriété est disponible dans le `CheckCtx` AST (elle est utilisée dans `dead_export/text.rs:32-34`).
