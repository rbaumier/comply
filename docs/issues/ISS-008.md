# no-extraneous-import et no-default-export : pas d'exception pour les fichiers de config

**Règles** : `no-extraneous-import`, `no-default-export`
**Impact** : 10 faux positifs sur Amadeo (7 + 3)
**Priorité** : moyenne

## Problème

### `no-extraneous-import` (7 FP)

Flagge les imports de devDependencies dans les fichiers de config racine :
- `import { defineConfig } from 'vite'` dans `vite.config.ts`
- `import { defineConfig } from '@playwright/test'` dans `playwright.config.ts`
- `import react from '@vitejs/plugin-react'` dans `vite.config.ts`

Ces fichiers ne sont jamais en production — ils sont exécutés par les outils de build qui ont accès aux devDependencies.

### `no-default-export` (3 FP)

Flagge `export default defineConfig(...)` dans les fichiers de config et les routes TanStack. Ces fichiers **doivent** utiliser des default exports — c'est une exigence du framework/outil.

## Fix proposé

Les deux règles devraient consulter `is_config_file()` (déjà implémenté dans `unused_file/text.rs:122-134`) et skip ces fichiers.

Pour `no-default-export`, ajouter aussi une exception pour les fichiers dans les dossiers de routes framework (via `framework_entry_dirs()`), puisque les default exports y sont imposés par le router.

Pattern de config files à exclure :
- `*.config.ts`, `*.config.js`, `*.config.mjs`
- Fichiers racine type `tailwind.config.*`, `postcss.config.*`, `drizzle.config.*`
- Fichiers matchant `is_config_file()` existant

## Fichiers concernés

- `src/rules/no_extraneous_import/` — ajouter exclusion config files
- `src/rules/no_default_export/` — ajouter exclusion config files + route dirs
- `src/rules/unused_file/text.rs:122-134` — `is_config_file()` à extraire comme utilitaire partagé
