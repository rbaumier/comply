# tailwind-classnames-order : doublon avec prettier-plugin-tailwindcss

**Règle** : `tailwind-classnames-order`
**Impact** : 69 faux positifs sur Amadeo (pas vraiment des FP, mais du bruit redondant)
**Priorité** : basse

## Problème

La règle impose un ordre des classes Tailwind CSS. Mais quand le projet utilise `prettier-plugin-tailwindcss`, Prettier reformate déjà les classes automatiquement au save/commit. Les diagnostics comply sont donc du bruit — le développeur ne peut rien y faire manuellement puisque Prettier va re-trier.

## Fix proposé

Détecter la présence de `prettier-plugin-tailwindcss` dans les devDependencies du `package.json`. Si présent, désactiver automatiquement `tailwind-classnames-order`.

Implémentation possible :
- Ajouter `prettier-plugin-tailwindcss` dans un TOML `tailwind.toml` avec un champ `suppresses_rules` (nouveau concept)
- Ou plus simple : vérifier dans le check de la règle via `ProjectCtx` si la dep est présente, et skip

## Fichiers concernés

- `src/rules/tailwind_classnames_order/` — ajouter la détection
- Potentiellement `src/frameworks/mod.rs` — si on ajoute un concept `suppresses_rules`
