# ts-no-extraneous-class : les classes décorées sont flaggées comme vides

**Règle** : `ts-no-extraneous-class`
**Type** : bug (faux positif)
**Priorité** : haute
**Source** : recettage post-merge worktree-fp-hunting

## Problème

La règle ne détecte pas les décorateurs. Une classe décorée comme `@Module({}) export class AppModule {}` est vide du point de vue du body, mais le décorateur porte la sémantique (metadata NestJS, Angular, etc.).

Le code (lignes 8-14) ne vérifie que `class_heritage` / `extends_clause` pour skip, jamais `decorator` :

```rust
for child in node.named_children(&mut cursor) {
    if child.kind() == "class_heritage" || child.kind() == "extends_clause" {
        return;
    }
}
```

Les tests confirment le comportement actuel (lignes 145-151) : les classes décorées SONT flaggées.

## Exemples FP

```ts
@Module({ imports: [UserModule] })
export class AppModule {}

@Injectable()
export class AuthGuard {}

@Component({ selector: 'app-root', template: '<router-outlet/>' })
export class AppComponent {}
```

## Fix proposé

Ajouter un check `decorator` dans la boucle named_children :

```rust
if child.kind() == "class_heritage" || child.kind() == "extends_clause"
    || child.kind() == "decorator"
{
    return;
}
```

Mettre à jour les tests `flags_decorated_empty_class` → `allows_decorated_empty_class`.

## Statut

✅ résolu — décorateurs détectés sur node et parent (export_statement)
