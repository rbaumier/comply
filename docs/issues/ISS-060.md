# Classification fichiers : .d.tsx et .d.cts non skippés

**Type** : bug (faux positifs)
**Priorité** : moyenne
**Source** : recettage post-merge worktree-fp-hunting

## Problème

`files.rs` skip les fichiers de déclaration TypeScript `.d.ts` et `.d.mts` (lignes 94, 237) mais omet deux extensions légitimes :

- `.d.tsx` — déclarations de composants React typés (rare mais valide, ex: `types/react-table.d.tsx`)
- `.d.cts` — déclarations CommonJS TypeScript (pendant de `.d.mts`)

```rust
// Actuel
if name.ends_with(".d.ts") || name.ends_with(".d.mts") {
    return None;
}
```

Ces fichiers ne contiennent aucune implémentation, uniquement des types. Les linter dessus produit du bruit (unused-import, no-extraneous-class, etc.).

## Fix proposé

```rust
if name.ends_with(".d.ts") || name.ends_with(".d.mts")
    || name.ends_with(".d.cts") || name.ends_with(".d.tsx")
{
    return None;
}
```

Aux deux emplacements (lignes 94 et 237).

## Statut

✅ résolu — .d.cts et .d.tsx ajoutés aux deux emplacements
