# playwright-no-page-pause : gating incohérent et fire sur tests non-Playwright

**Règle** : `playwright-no-page-pause`
**Type** : bug (faux positif + incohérence)
**Priorité** : moyenne
**Source** : recettage post-merge worktree-fp-hunting

## Problème

### 1. Pattern de gating différent des 35 autres règles

Les 35 autres règles playwright utilisent le pattern standard :
```rust
if !source.windows(16).any(|w| w == b"@playwright/test") { return; }
```

`playwright-no-page-pause` utilise un pattern différent (lignes 13-18) :
```rust
let imports_playwright = source
    .windows(b"from \"@playwright/test\"".len())
    .any(|w| w == b"from \"@playwright/test\"")
    || source
        .windows(b"from '@playwright/test'".len())
        .any(|w| w == b"from '@playwright/test'");
```

Ce pattern est plus strict (exige `from "..."`) mais aussi plus fragile (rate les `require("@playwright/test")`).

### 2. Fallback is_test_file fire sur tests non-Playwright

Ligne 19 :
```rust
if !imports_playwright && !is_test_file(ctx.path) { return; }
```

Si `imports_playwright` est false mais `is_test_file` est true, la règle fire quand même. Cela signifie que tout fichier `.test.ts` ou `.spec.ts` est ciblé, même sans Playwright. Un fichier Jest avec une variable `page` qui a une méthode `pause()` sera flaggé.

## Fix proposé

Aligner sur le pattern standard des 35 autres règles, sans fallback `is_test_file` :

```rust
if !source.windows(16).any(|w| w == b"@playwright/test") { return; }
```

Supprimer `is_test_file` et `TEST_MARKERS` qui deviennent inutiles.

## Statut

✅ résolu — aligné sur le pattern standard 16-byte window

## Vérification

- [ ] Un fichier `.test.ts` sans import `@playwright/test` contenant `page.pause()` n'est PAS flaggé
- [ ] Un fichier `.ts` avec `import { test } from "@playwright/test"` contenant `page.pause()` EST flaggé
- [ ] Pattern aligné avec les 35 autres règles playwright
