# ts-no-magic-numbers fire sur fichiers Rust

**Règle** : `ts-no-magic-numbers`
**Impact** : 398 faux positifs sur tokio
**Priorité** : haute

## Problème

La règle `ts-no-magic-numbers` est destinée aux fichiers TypeScript/JavaScript mais fire sur des fichiers `.rs`. Exemples concrets dans tokio :

```rust
// tokio-stream/src/iter.rs:51
if self.yield_amt >= 32 {
// → flaggé comme "Magic number `32`"
```

En Rust, les constantes numériques dans les comparaisons de capacité, les tailles de buffer, les seuils internes sont standards et ne méritent pas d'être extraites en `const` nommée dans tous les cas.

## Fix proposé

Vérifier le scope de la règle : `ts-no-magic-numbers` ne doit s'appliquer qu'aux fichiers `.ts`, `.tsx`, `.js`, `.jsx`. Si un backend Rust existe pour cette règle, il doit être séparé avec ses propres seuils.

## Fichiers concernés

- `src/rules/ts_no_magic_numbers/` — vérifier le filtre d'extension
- Potentiellement `src/engine/mod.rs` — vérifier le dispatch par langage
