# ts-no-loop-func fire sur closures Rust

**Règle** : `ts-no-loop-func`
**Impact** : 135 faux positifs sur tokio
**Priorité** : haute

## Problème

La règle flagge les closures déclarées à l'intérieur de boucles. En JavaScript, c'est problématique car les closures capturent par référence et sont allouées sur le heap. En Rust, les closures sont zero-cost, inline par le compilateur, et la capture est explicite (move/borrow).

Exemple tokio :
```rust
// tokio-stream/tests/stream_stream_map.rs:419
let n = poll_fn(|cx| stream_map.poll_next_many(cx, &mut buffer, 3)).await;
```

Ce pattern est totalement idiomatique en Rust.

## Fix proposé

`ts-no-loop-func` ne doit pas s'appliquer aux fichiers `.rs`. Si une variante Rust est souhaitée un jour, elle devrait se concentrer sur les closures qui font des allocations heap (`Box::new`, `Arc::new`) dans une boucle hot path.

## Fichiers concernés

- `src/rules/ts_no_loop_func/` — vérifier le filtre d'extension
