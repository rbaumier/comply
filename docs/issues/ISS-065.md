# no-history-in-comments FP sur "was"/"previously" descriptif

**Règle** : `no-history-in-comments`
**Impact** : 406 faux positifs sur tokio (majorité sont des FP)
**Priorité** : haute

## Problème

La règle flagge les mots `was`, `previously`, `refactored`, `rewritten` dans les commentaires, partant du principe qu'ils narrent l'historique. Mais ces mots sont fréquemment utilisés dans un sens descriptif/conditionnel :

```rust
// "Return to the previously set panic hook" → description d'état, pas d'historique
// "whether it was the initial value or sent afterwards" → conditionnel dans docstring
// "Track which source stream the value was received from" → voix passive descriptive
```

Le mot "was" en anglais est le passé de "to be" et apparaît dans énormément de phrases descriptives. Le pattern actuel est beaucoup trop large.

## Fix proposé

Affiner l'heuristique :
1. Ignorer "was" quand il est dans une clause conditionnelle (`whether...was`, `if...was`)
2. Ignorer "previously" quand suivi d'un nom (= "le X précédent") vs quand suivi d'un verbe (= "auparavant on faisait")
3. Ignorer les mots dans les doc comments `///` et `//!` qui décrivent l'API, pas l'historique
4. Se concentrer sur les patterns réellement historiques : "was changed to", "previously used", "was refactored from", "rewritten because"
