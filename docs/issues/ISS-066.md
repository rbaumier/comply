# comment-prose-quality : "lexical illusion" et prose checks sur code examples

**Règle** : `comment-prose-quality`
**Impact** : 1038 faux positifs sur tokio
**Priorité** : haute

## Problème

Trois sous-problèmes :

### 1. "Lexical illusion: `}` repeated" sur code examples

La règle détecte des mots/symboles répétés sur des lignes consécutives. Mais dans les doc comments Rust (`///`), les blocs de code contiennent naturellement des `}` de fermeture sur des lignes successives :

```rust
///     }
/// }
///
// → flaggé comme "lexical illusion"
```

### 2. "Weasel word `many`" dans API docs

Le mot "many" est parfaitement approprié dans la documentation d'API ("handles many concurrent connections"). Forcer la précision dans les docstrings d'API est trop pédantique.

### 3. "Passive voice `is used`" dans API docs

La voix passive est le standard dans la documentation technique d'API ("this method is used to...", "the buffer is flushed when...").

## Fix proposé

1. Exclure les blocs de code dans les doc comments (lignes entre ` ``` ` ou indentées de 4+ espaces dans `///`)
2. Relâcher "weasel word" et "passive voice" dans les doc comments (`///`, `//!`) — ces checks sont pertinents pour les commentaires inline mais pas pour la documentation d'API
3. Ne flagguer "lexical illusion" que sur du texte en prose, pas sur des symboles de code (`{`, `}`, `)`, `]`)
