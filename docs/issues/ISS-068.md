# boolean-naming FP sur noms idiomatiques Rust

**Règle** : `boolean-naming`
**Impact** : 84 faux positifs sur tokio
**Priorité** : moyenne

## Problème

La règle exige un préfixe prédicat (`is_*`, `has_*`, `should_*`...) sur tout booléen. Mais certains noms sont idiomatiques en Rust et se suffisent à eux-mêmes :

- `done` — universellement compris, `is_done` est verbeux sans gain
- `success` — clair en soi
- `called` — dans les tests, `was_called` est acceptable mais `called` est standard
- `polled` — contexte Rust async, parfaitement clair
- `had_error` — a déjà un préfixe passé mais la règle ne le reconnaît pas
- `stall` — nom technique dans le contexte d'un runtime

## Fix proposé

Ajouter une whitelist de noms booléens idiomatiques en Rust :
- `done`, `success`, `found`, `valid`, `ready`, `closed`, `connected`, `available`, `empty`, `alive`
- Reconnaître `had_*` comme un préfixe valide (c'est un passé composé, donc un prédicat)

## Fichiers concernés

- `src/rules/boolean_naming/` — ajouter whitelist Rust et reconnaître `had_*`
