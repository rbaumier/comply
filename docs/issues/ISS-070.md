# Nouvelle règle : rust-no-allow-without-reason

**Règle proposée** : `rust-no-allow-without-reason`
**Type** : nouvelle règle
**Priorité** : moyenne

## Motivation

Les attributs `#[allow(...)]` désactivent des warnings du compilateur ou de Clippy. Sans commentaire de justification, ils masquent des problèmes légitimes et deviennent de la dette technique silencieuse.

Exemples trouvés dans tokio :
```rust
#[allow(dead_code)]           // pourquoi est-ce mort ?
#[allow(clippy::needless_lifetimes)]  // pourquoi clippy a tort ici ?
#[allow(unused_imports)]      // pourquoi garder un import inutilisé ?
```

Certains sont justifiés (ex: `#[allow(clippy::wrong_self_convention)] // mirrors std name`) — la règle ne doit flagguer que ceux sans commentaire.

## Spécification

- Flagguer tout `#[allow(...)]` qui n'a pas de commentaire `//` sur la même ligne ou la ligne précédente
- Exclure `#[allow(unused)]` dans les `#[cfg(test)]` modules (pattern standard)
- Exclure `#[allow(dead_code)]` sur les items `#[cfg(...)]` (code conditionnel)
- Severity : `warning`

- Nouveau : `src/rules/rust_no_allow_without_reason/` — règle AST sur tree-sitter Rust
