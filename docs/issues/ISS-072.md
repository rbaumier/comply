# api-response-envelope-consistency crash sur emoji multi-byte (UTF-8)

**Règle** : `api-response-envelope-consistency`
**Impact** : crash (panic) — bloque l'analyse complète du projet
**Priorité** : critique

## Problème

La règle panic sur les caractères multi-byte (emoji) avec l'erreur :

```
byte index 62 is not a char boundary; it is inside '💡' (bytes 61..65)
```

Le code à `typescript.rs:114:24` indexe un `&str` par position byte au lieu de position caractère. L'emoji 💡 fait 4 bytes (UTF-8) mais le code suppose 1 byte par caractère.

Reproduit sur trpc :
```typescript
// 💡 Tip: Try adding a new property here
```

Ce crash empêche l'analyse complète du projet trpc (0 diagnostics produits).

## Fix proposé

Remplacer l'indexation byte (`&text[..n]` ou `text.as_bytes()[n]`) par une indexation char-aware. Options :
1. `text.char_indices().nth(n)` pour obtenir la bonne position
2. `text.get(..n)` qui retourne `Option` au lieu de panic
3. Utiliser les positions byte du tree-sitter AST (qui sont correctes) sans recalcul

## Fichiers concernés

- `src/rules/api_response_envelope_consistency/typescript.rs:114` — fix l'indexation
