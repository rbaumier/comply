# no-undefined-argument FP dans les matchers d'assertion

**Règle** : `no-undefined-argument`
**Impact** : 57 faux positifs sur zustand
**Priorité** : moyenne

## Problème

La règle interdit de passer `undefined` comme argument, suggérant d'omettre l'argument. Mais dans les assertions de test, `undefined` est une valeur attendue intentionnelle :

```typescript
expect(onRehydrateStorageSpy).toHaveBeenCalledWith(
  { count: 42, name: 'test-storage' },
  undefined,  // ← vérifie explicitement que le 2e arg est undefined (pas d'erreur)
)
```

Omettre `undefined` ici changerait la sémantique de l'assertion : `toHaveBeenCalledWith(state)` vérifie un seul argument, pas deux.

## Fix proposé

Exclure `undefined` comme argument quand il est passé à un matcher vitest/jest :
- `.toHaveBeenCalledWith(..., undefined)`
- `.toEqual(undefined)`
- `.toBe(undefined)`
- Tout appel de matcher dans un `expect()`

Alternative : exclure `undefined` quand il n'est pas le dernier argument (car il ne peut pas être omis sans changer la position des arguments suivants).
