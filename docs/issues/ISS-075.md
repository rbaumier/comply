# unused-component-prop FP sur fichiers de type-test

**Règle** : `unused-component-prop`
**Impact** : 82 faux positifs sur zustand
**Priorité** : moyenne

## Problème

Les fichiers `types.test.tsx` vérifient l'inférence de types TypeScript — les props sont déclarées pour valider que le système de types les accepte, mais elles ne sont pas lues au runtime.

```typescript
// types.test.tsx — teste que les types sont inférés correctement
num: number           // ← déclaré mais pas lu → FP
numGet: () => number  // ← idem
numSet: (v: number) => void
```

Flagguer ces props comme "inutilisées" est un FP car le fichier n'a pas vocation à utiliser les valeurs.

## Fix proposé

Ne pas appliquer `unused-component-prop` sur les fichiers dont le nom matche :
- `types.test.*` / `*.types.test.*`
- `*.d.ts` (fichiers de déclaration)
- Fichiers contenant `@vitest-environment` ou `// @ts-check` type-level testing patterns

## Fichiers concernés

- `src/rules/unused_component_prop/` — ajouter exclusion par nom de fichier
