# no-property-mutation trop strict dans les fichiers de test

**Règle** : `no-property-mutation`
**Impact** : 106 faux positifs sur zustand
**Priorité** : moyenne

## Problème

La règle flagge toute mutation de propriété. Mais dans les tests, certaines mutations sont des patterns standard :

### 1. Monkey-patching console/globals pour test setup
```typescript
console.error = consoleError  // → test teardown, restaure le handler
```

### 2. State mutation dans les tests de state managers
```typescript
// zustand est un state manager — tester la mutation EST le but du test
```

### 3. Mock setup
```typescript
window.localStorage = mockStorage  // → mock de l'environnement
```

## Fix proposé

Relâcher `no-property-mutation` dans les fichiers de test (`.test.ts`, `.test.tsx`, `.spec.ts`) pour :
1. Les mutations de `console.*`, `window.*`, `global.*`, `globalThis.*` (monkey-patching d'environnement)
2. Les mutations dans les blocs `beforeEach` / `afterEach` / `beforeAll` / `afterAll`
3. Ou réduire la sévérité à `info` dans les fichiers de test

## Fichiers concernés

- `src/rules/no_property_mutation/` — ajouter exceptions pour les fichiers de test
