# consistent-function-scoping FP dans les callbacks de test

**Règle** : `consistent-function-scoping`
**Impact** : 44 faux positifs sur zustand
**Priorité** : basse

## Problème

La règle suggère de hoist les fonctions qui ne capturent pas de variable du scope parent. Mais dans les tests, définir des helpers localement dans le `it()` est un pattern de lisibilité :

```typescript
it('handles unstorable methods', () => {
  const unstorableMethod = () => {}  // ← intentionnellement local au test
  const useBoundStore = create(
    persist(() => ({ count: 0, unstorableMethod }), { ... })
  )
})
```

Hoister `unstorableMethod` hors du test :
- Réduit la lisibilité (le lecteur doit chercher la définition)
- Pollue le scope module avec du setup spécifique à un test
- N'apporte aucun gain de performance (les tests ne sont pas hot paths)

## Fix proposé

Ne pas appliquer `consistent-function-scoping` dans les blocs `it()`, `test()`, `describe()` des fichiers de test. Les fonctions définies localement dans un test sont du setup intentionnel, pas un problème de scoping.

## Fichiers concernés

- `src/rules/consistent_function_scoping/` — exclure les blocs de test
