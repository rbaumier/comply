# no-implicit-deps ne résout pas les dépendances workspace (monorepo)

**Règle** : `no-implicit-deps`
**Impact** : 806 faux positifs sur redux-toolkit
**Priorité** : haute

## Problème

La règle vérifie que les imports bare (`import X from 'pkg'`) sont listés dans `package.json`. Mais dans un monorepo (yarn workspaces, pnpm workspaces, npm workspaces), les dépendances sont souvent :

1. Déclarées dans le `package.json` du **workspace** (ex: `website/package.json`), pas dans le root
2. Hoistées par le package manager dans le `node_modules` root
3. Fournies par le framework (ex: `@docusaurus/preset-classic` est un peer dep implicite de Docusaurus)

Exemples sur RTK :
```
website/docusaurus.config.ts → import '@docusaurus/preset-classic'
// → flaggé car pas dans root package.json, mais dans website/package.json
```

## Fix proposé

1. Pour chaque fichier importateur, remonter l'arborescence jusqu'au plus proche `package.json` (pas seulement le root)
2. Vérifier les `dependencies`, `devDependencies`, ET `peerDependencies` de ce package.json
3. Si un workspace root est détecté (`workspaces` field), vérifier aussi les dépendances des packages ancêtres
4. Reconnaître les virtual modules de frameworks (ex: `@theme/`, `@docusaurus/`, `virtual:`)

## Fichiers concernés

- `src/rules/no_implicit_deps/` — résolution workspace-aware des dépendances
