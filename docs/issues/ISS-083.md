# explicit-units FP sur noms standard Rust (length, offset, width)

**Règle** : `explicit-units`
**Impact** : 804 faux positifs sur polars
**Priorité** : moyenne

## Problème

La règle veut un suffixe d'unité (`_ms`, `_bytes`, `_count`) sur tout nom numérique "ambigu". Mais en Rust, `length`, `offset`, `width` ont une signification universelle dans le contexte des buffers, slices, et structures de données :

```rust
pub fn zeroed(length: usize) -> Self {
    // → flaggé "ambiguous length — add _bytes or _count"
    // Mais length: usize dans un Buffer<T> = nombre d'éléments (convention Rust)
```

En Rust :
- `length` / `len` = nombre d'éléments (convention `.len()` de la stdlib)
- `offset` = décalage en éléments (convention `ptr::offset`)
- `width` = nombre de colonnes/champs (convention data)

Le type `usize` et le contexte rendent l'unité évidente. Forcer `length_count` est du bruit.

## Fix proposé

1. Ne pas flagguer `length`/`len`, `offset`, `width`, `height`, `size`, `capacity` quand le type est `usize` ou `u64` (en Rust, ces types impliquent "éléments" par convention)
2. Ou ne flagguer que quand l'ambiguïté est réelle (ex: un paramètre `timeout` qui pourrait être en ms ou en secondes)
3. Concentrer la règle sur les temps (`timeout`, `delay`, `duration`, `interval`) et les tailles de données (`size` quand bytes/éléments est ambigu)
