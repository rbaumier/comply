# rust-unused-dep ne gère pas les dépendances feature-gated

**Règle** : `rust-unused-dep`
**Impact** : 1665 faux positifs sur polars
**Priorité** : haute

## Problème

Polars utilise massivement les feature flags Cargo. De nombreuses dépendances sont `optional = true` et ne sont utilisées que derrière `#[cfg(feature = "...")]` :

```toml
# Cargo.toml
uuid = { workspace = true, features = ["js"] }
bincode = { workspace = true, optional = true }
rand = { workspace = true, optional = true }
```

```rust
#[cfg(feature = "concat_str")]
pub(super) fn join(...) { ... }  // utilise une dep feature-gated
```

La règle analyse les `use`/`extern crate` dans le code source mais ne voit que le code compilé par défaut (sans les feature flags activés). Les deps optionnelles apparaissent donc comme "inutilisées".

## Fix proposé

1. Ne pas flagguer les dépendances marquées `optional = true` dans `Cargo.toml`
2. Pour les deps non-optionnelles, scanner aussi le code derrière `#[cfg(feature = "...")]`
3. Alternative simple : ignorer `rust-unused-dep` pour les crates avec plus de N features définies (heuristique "crate modulaire")

## Fichiers concernés

- `src/rules/rust_unused_dep/` — gérer `optional = true` et `#[cfg(feature)]`
