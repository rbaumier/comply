# assertions-in-tests / playwright-expect-expect FP sur testing-library

**Règles** : `assertions-in-tests`, `playwright-expect-expect`
**Impact** : 175 + 177 = 352 faux positifs sur swr
**Priorité** : haute

## Problème

Les deux règles exigent des assertions `expect(...)` dans les tests. Mais les tests @testing-library/react utilisent des assertions implicites :

```typescript
it('should respect requests after key has changed', async () => {
  renderWithConfig(<Page />)
  screen.getByText('data:')                    // ← assertion implicite (throws si absent)
  await screen.findByText('data:short request') // ← assertion async (throws si timeout)
})
```

`screen.getByText()` et `screen.findByText()` sont des **assertions** — elles throw si l'élément n'est pas trouvé. Mais les règles ne reconnaissent que `expect()`.

De plus, `playwright-expect-expect` est un nom trompeur : elle fire sur des tests non-Playwright (vitest + testing-library ici).

## Fix proposé

1. Reconnaître les queries testing-library comme des assertions : `getByText`, `getByRole`, `getByTestId`, `findByText`, `findByRole`, `queryByText` (quand suivi de `.toBe(null)`)
2. Reconnaître `waitFor()` comme un wrapper d'assertion
3. `playwright-expect-expect` ne devrait fire QUE sur les fichiers Playwright (détection via import de `@playwright/test`)
4. Ou fusionner les deux règles en une seule `test-has-assertion` qui comprend tous les frameworks

## Fichiers concernés

- `src/rules/assertions_in_tests/` — ajouter testing-library queries
- `src/rules/playwright_expect_expect/` — scoper aux fichiers Playwright
