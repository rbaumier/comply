# no-test-return-statement FP sur return dans les fonctions imbriquées

**Règle** : `no-test-return-statement`
**Impact** : 377 faux positifs sur swr
**Priorité** : haute

## Problème

La règle interdit `return` dans les tests. Mais elle flagge les `return` dans des **fonctions imbriquées** à l'intérieur du test, pas dans le callback de test lui-même :

```typescript
it('should respect requests', async () => {
  const { data } = useSWR(key, () => {
    if (mounted) {
      return createResponse('short', { delay: 50 })  // ← flaggé à tort
    }
    return createResponse('long', { delay: 100 })     // ← flaggé à tort
  })
})
```

Le `return` est dans le **fetcher callback** passé à `useSWR`, pas dans le test. Le test ne retourne rien — ce sont les closures/callbacks internes qui le font.

## Fix proposé

Ne flagguer `return` que dans la **fonction de test directe** (le callback de `it()` / `test()`), pas dans les fonctions imbriquées à l'intérieur. Vérifier la profondeur de nesting : seuls les `return` au premier niveau de la fonction de test sont suspects.

## Fichiers concernés

- `src/rules/no_test_return_statement/` — vérifier la profondeur de nesting du return
