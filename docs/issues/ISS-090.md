# Règles UI/a11y/tailwind ne devraient pas fire sur le JSX de test

**Règles** : `tailwind-require-focus-ring`, `a11y-click-events-have-key-events`, `react-button-has-type`, `html-require-button-type`
**Impact** : 131 + 183 + 120 + 120 = 554 faux positifs sur swr
**Priorité** : haute

## Problème

Les règles d'accessibilité, Tailwind, et HTML quality firent sur le JSX dans les fichiers de test. Mais ce JSX est du **fixture de test** — il n'est pas livré aux utilisateurs :

```typescript
// test/use-swr-key.test.tsx
return <button onClick={() => setMounted(s => s + 1)}>click</button>
// → flaggé: missing focus:ring-*, missing onKeyDown, missing type="button"
```

Ces boutons existent uniquement pour tester la logique de swr, pas pour être accessibles. Exiger un focus ring, un handler clavier, et un type sur chaque `<button>` de test est du bruit pur.

## Fix proposé

Exclure les fichiers de test (`*.test.tsx`, `*.test.ts`, `*.spec.tsx`, `__tests__/`) de toutes les règles UI/a11y/Tailwind :

Règles concernées (au minimum) :
- `tailwind-require-focus-ring`
- `a11y-click-events-have-key-events`
- `react-button-has-type` / `html-require-button-type`
- `a11y-anchor-is-valid`
- `a11y-img-alt-text`
- Toutes les règles `a11y-*`, `tailwind-*` sur les fichiers de test

Alternative : ajouter un flag global `skip_test_files` pour les catégories `a11y` et `tailwind`.

## Fichiers concernés

- `src/engine/mod.rs` ou `src/rules/mod.rs` — filtre global test-files pour catégories a11y/tailwind
