# import-no-commonjs / prefer-module doivent respecter package.json "type"

**Règles** : `import-no-commonjs`, `no-require-imports`, `prefer-module`
**Impact** : 3095 faux positifs sur fastify (1036 + 972 + 1087)
**Priorité** : haute

## Problème

Les règles imposent ESM (`import`/`export`) et interdisent CJS (`require`/`module.exports`). Mais fastify utilise **intentionnellement** CommonJS :

```javascript
'use strict'
const http = require('http')     // → flaggé par import-no-commonjs
module.exports = fastify         // → flaggé par prefer-module
```

Le `package.json` de fastify n'a PAS `"type": "module"`, ce qui signifie que CJS est le format voulu. Forcer ESM sur un projet CJS est un FP massif.

## Fix proposé

Vérifier le `package.json` le plus proche du fichier :
1. Si `"type": "module"` → appliquer les règles ESM
2. Si `"type": "commonjs"` ou absent → ne PAS appliquer `import-no-commonjs`, `no-require-imports`, `prefer-module`
3. Si le fichier a l'extension `.mjs` → ESM, `.cjs` → CJS (indépendant du package.json)

## Fichiers concernés

- `src/rules/import_no_commonjs/`, `no_require_imports/`, `prefer_module/` — ajouter détection du module system
