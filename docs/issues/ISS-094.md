# data-clumps FP sur les signatures d'API framework

**Règle** : `data-clumps`
**Impact** : 541 faux positifs sur fastify
**Priorité** : moyenne

## Problème

La règle détecte des groupes de paramètres qui apparaissent ensemble dans de nombreuses fonctions et suggère de les extraire dans un type. Mais sur fastify, la signature `(instance, opts, done)` est définie par le **contrat d'API du framework** :

```javascript
fastify.register((instance, opts, done) => {
  // → flaggé: "Parameters [done, instance, opts] appear together in 46 functions"
```

Le développeur ne peut pas "extraire dans un type" — c'est la signature imposée par fastify pour les plugins. Idem pour `(req, reply, done)` qui est la signature des hooks.

## Fix proposé

1. Exclure les paramètres de callbacks passés à des fonctions de framework connues (`register`, `addHook`, `route`, `get`, `post`, etc.)
2. Alternative : ne flagguer que si les paramètres répétés sont dans des fonctions **déclarées par l'utilisateur** (non des callbacks)
3. Ne pas compter les paramètres dans les fichiers de test — les tests répètent souvent les mêmes patterns par design

## Fichiers concernés

- `src/rules/data_clumps/` — exclure les signatures de callback framework
