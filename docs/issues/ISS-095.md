# require-hook flagge les require() d'import comme side effects

**Règle** : `require-hook`
**Impact** : 986 faux positifs sur fastify
**Priorité** : moyenne

## Problème

La règle exige que les side effects dans les fichiers de test soient dans des hooks (`beforeEach`, `beforeAll`). Mais elle flagge les `require()` d'import de modules au top-level :

```javascript
// test/reply-code.test.js
'use strict'
const { test } = require('node:test')      // ← flaggé comme side effect
const { Readable } = require('node:stream') // ← flaggé
const Fastify = require('..')               // ← flaggé
```

Les `require()` sont des **imports**, pas des side effects. Ils sont l'équivalent CJS de `import`. Les mettre dans `beforeEach` serait absurde — on ne ré-importe pas un module à chaque test.

## Fix proposé

Ne pas flagguer les `require()` qui sont des imports de module au top-level :
1. `const X = require('...')` → import, pas un side effect
2. `const { X } = require('...')` → destructured import
3. Ne flagguer que les appels de fonction arbitraires au top-level (ex: `setupDatabase()`, `server.listen()`)

## Fichiers concernés

- `src/rules/require_hook/` — exclure les patterns d'import CJS
