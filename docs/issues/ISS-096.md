# filename-naming-convention FP sur les conventions SvelteKit (+page, +server)

**Règle** : `filename-naming-convention`
**Impact** : 601 faux positifs sur svelte-kit
**Priorité** : haute

## Problème

SvelteKit impose des noms de fichiers avec le préfixe `+` pour le routing :
- `+page.svelte` — page component
- `+page.server.ts` — server-side load function
- `+layout.svelte` — layout component
- `+server.ts` — API endpoint
- `+error.svelte` — error page

La règle flagge tous ces fichiers car `+page.server.ts` ne match pas kebab-case. C'est un FP majeur — ce sont des conventions imposées par le framework, pas des choix du développeur.

Aussi impacté : les dynamic routes Next.js (`[slug]`, `[...rest]`) qui contiennent des brackets dans le nom de fichier.

## Fix proposé

1. Reconnaître les conventions de routing par framework :
   - SvelteKit : `+page.svelte`, `+layout.svelte`, `+server.ts`, `+error.svelte`, `+page.server.ts`
   - Next.js : `[param]`, `[...rest]`, `(group)`, `page.tsx`, `layout.tsx`, `loading.tsx`
   - Nuxt : `[id].vue`, `[...slug].vue`
2. Utiliser la framework detection existante pour appliquer les exceptions
