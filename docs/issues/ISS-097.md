# jsdoc-needs-description FP sur les annotations JSDoc type-only

**Règle** : `jsdoc-needs-description`
**Impact** : 2181 faux positifs sur svelte-kit
**Priorité** : haute

## Problème

Dans l'écosystème JavaScript (surtout Svelte, qui était JS-first), JSDoc est utilisé pour les **annotations de type**, pas pour la documentation :

```javascript
/** @type {import('@sveltejs/kit').Config} */
const config = { ... }
```

Ce pattern est l'équivalent JS de `const config: Config = { ... }` en TypeScript. Le bloc JSDoc n'a qu'un `@type` tag — exiger une "prose description" n'a pas de sens, car le bloc ne documente pas une API, il type une variable.

Ce pattern est recommandé par la documentation officielle de Svelte, TypeScript (via `// @ts-check`), et VS Code.

## Fix proposé

Ne pas flagguer les blocs JSDoc qui ne contiennent que des tags de typage :
1. `@type` seul → annotation de type, pas de description nécessaire
2. `@param` + `@returns` sans autre contenu → signature de type
3. Ne flagguer que les blocs JSDoc qui ont des tags de documentation (`@example`, `@see`, `@deprecated`, `@throws`) sans description — là, la description manquante est un vrai oubli
