# node-no-sync FP sur les scripts de build et CLI

**Règle** : `node-no-sync`
**Impact** : 317 faux positifs sur svelte-kit
**Priorité** : moyenne

## Problème

La règle interdit les méthodes synchrones Node.js (`existsSync`, `readFileSync`, `execSync`). C'est pertinent pour le code serveur (bloque l'event loop) mais pas pour :

1. **Scripts de build** (`scripts/`, `bin/`) — exécutés une seule fois, blocking est OK
2. **Fichiers de config** (`svelte.config.js`, `vite.config.ts`, `webpack.config.js`) — exécutés au build, pas en runtime
3. **CLI tools** — séquentiels par nature

Exemples flaggés à tort :
```javascript
// scripts/check-dependencies.js
const result = execSync('npx check-dependencies')  // → flaggé
// scripts/sync-all.js
const dirs = fs.readdirSync(path)                   // → flaggé
```

## Fix proposé

Exclure `node-no-sync` pour les fichiers dans :
1. `scripts/`, `bin/`, `tools/`, `cli/`
2. `*.config.js`, `*.config.ts`, `*.config.mjs`
3. Fichiers avec un shebang `#!/usr/bin/env node` en première ligne

## Fichiers concernés

- `src/rules/node_no_sync/` — ajouter exclusions par chemin et type de fichier
