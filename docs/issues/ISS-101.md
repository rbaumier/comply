# Stack overflow sur les projets volumineux (playwright)

**Impact** : crash (stack overflow) — comply ne peut pas analyser le projet
**Priorité** : haute

## Problème

Sur le projet playwright (~4700 fichiers), comply crash avec :

```
thread '<unknown>' has overflowed its stack
```

Pas de backtrace exploitable. Le crash survient probablement dans le moteur de résolution de dépendances ou dans le graphe d'imports, qui utilise des récursions profondes sur les gros projets.

Playwright a une structure profonde de sous-packages (`packages/playwright-core/`, `packages/playwright-test/`, etc.) avec des milliers de fichiers TypeScript interconnectés.

## Fix proposé

1. Augmenter la taille par défaut de la stack des threads de travail (`RUST_MIN_STACK` ou `std::thread::Builder::stack_size`)
2. Convertir les récursions profondes (graph traversal, import resolution) en itérations avec stack explicite
3. Ajouter un guard de profondeur de récursion avec un warning plutôt qu'un crash

## Fichiers concernés

- `src/engine/mod.rs` — configuration de la taille de stack des threads
- `src/files.rs` — résolution de dépendances / graphe d'imports
