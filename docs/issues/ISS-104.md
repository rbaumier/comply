# ts-class-methods-use-this FP sur méthodes décorées (controllers/services)

**Règle** : `ts-class-methods-use-this`
**Impact** : ~734 faux positifs sur nest (production code seul)
**Priorité** : haute

## Problème

La règle recommande de rendre `static` ou d'extraire les méthodes qui n'utilisent pas `this`. Mais dans les frameworks à décorateurs (NestJS, Angular), les méthodes DOIVENT être des méthodes d'instance pour que les décorateurs fonctionnent :

```typescript
// integration/microservices/src/grpc/grpc.controller.ts
@Controller()
export class GrpcController {
  @GrpcMethod('Math')
  async sum({ data }: { data: number[] }): Promise<any> {
    //   ↑ flaggé "does not use this — make it static"
    return of({ result: data.reduce((a, b) => a + b) });
  }
}
```

Rendre `sum` static casserait le routing gRPC de NestJS. Le décorateur `@GrpcMethod` a besoin d'une méthode d'instance pour fonctionner.

Même pattern pour `@Get()`, `@Post()`, `@MessagePattern()`, `@EventPattern()`, `@SubscribeMessage()`, etc.

## Fix proposé

Ne pas flagguer les méthodes qui ont un décorateur, ou les méthodes dans une classe décorée :
1. Si la méthode a un décorateur (`@Get`, `@Post`, `@GrpcMethod`, etc.) → skip
2. Ou si la classe parente a un décorateur (`@Controller`, `@Injectable`) → skip toutes les méthodes

## Fichiers concernés

- `src/rules/ts_class_methods_use_this/` — skip méthodes/classes décorées
