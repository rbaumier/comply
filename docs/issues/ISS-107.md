# Toutes les règles playwright-* fire sur les tests vitest/jest

**Règles** : `playwright-no-conditional-in-test`, `playwright-prefer-strict-equal`, `playwright-max-expects`, `playwright-prefer-to-be`, `playwright-no-useless-await`, `playwright-missing-await`, `playwright-expect-expect`, `playwright-no-conditional-expect`
**Impact** : ~332 faux positifs sur jotai (qui n'utilise pas Playwright)
**Priorité** : haute
**Lié à** : ISS-088 (même famille mais scope plus large)

## Problème

ISS-088 couvrait `playwright-expect-expect` spécifiquement. Le problème est en fait que TOUTES les règles préfixées `playwright-*` fire sur n'importe quel fichier de test, même quand le projet utilise vitest, jest, ou mocha.

Jotai utilise vitest + @testing-library/react. Aucune dépendance Playwright dans le package.json. Pourtant 8 règles Playwright différentes fire sur 332 emplacements.

Détail :
| Règle | FPs |
|---|---|
| playwright-no-conditional-in-test | 101 |
| playwright-prefer-strict-equal | 85 |
| playwright-max-expects | 71 |
| playwright-prefer-to-be | 39 |
| playwright-no-useless-await | 15 |
| playwright-missing-await | 10 |
| playwright-expect-expect | 7 |
| playwright-no-conditional-expect | 4 |

## Fix proposé

Les règles `playwright-*` ne devraient fire QUE quand Playwright est détecté comme dépendance du projet :
1. Vérifier `package.json` pour `@playwright/test` dans les devDependencies
2. Ou détecter les imports `from '@playwright/test'` dans le fichier
3. Sans cette détection, skip toutes les règles playwright-*

## Fichiers concernés

- `src/frameworks/mod.rs` — détection Playwright comme framework
- `src/rules/playwright_*/` — toutes les règles playwright doivent gater sur la détection framework
