# function-component-definition FP sur composants de test inline

**Règle** : `function-component-definition`
**Impact** : ~301 faux positifs sur jotai (tests uniquement)
**Priorité** : moyenne

## Problème

La règle exige des `function` declarations plutôt que des arrow functions pour les composants React. Mais dans les fichiers de test, les composants sont des fixtures inline temporaires définis à l'intérieur de `it()` ou `describe()` :

```tsx
// tests/react/abortable.test.tsx
it('can abort with signal.aborted', async () => {
  const Component = () => {         // ← flaggé "should be a function declaration"
    const count = useAtomValue(derivedAtom)
    return <div>count: {count}</div>
  }
  render(<Component />)
})
```

Ces composants :
- Sont des fixtures locales, pas des composants exportés
- Sont souvent définis dans un scope de fonction (impossible d'utiliser `function` declaration hoistée)
- Sont utilisés pour tester le comportement d'un hook, pas pour être réutilisés

## Fix proposé

Ne pas flagguer `function-component-definition` dans les fichiers de test (`*.test.tsx`, `*.spec.tsx`, fichiers dans `__tests__/`).

## Fichiers concernés

- `src/rules/function_component_definition/` — skip fichiers de test
