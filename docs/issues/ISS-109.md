# Comply timeout/hang sur projets >1000 fichiers TS

**Impact** : comply ne termine pas — aucun résultat produit
**Priorité** : haute
**Lié à** : ISS-101 (stack overflow est un symptôme du même problème)

## Problème

Comply ne termine pas (ou prend >60s) sur des projets TypeScript de taille moyenne à grande :

| Projet | Fichiers TS/JS | Résultat |
|---|---|---|
| nextjs | 21868 | timeout >60s, 0 output |
| playwright | 4700 | stack overflow |
| remix | 1301 | timeout >60s, 0 output |
| effect | 1770 | à tester |
| nest | 1673 | OK (~30s) |
| jotai | 177 | OK (~5s) |

Le seuil semble être autour de 1300-1700 fichiers, mais nest (1673) fonctionne. Le facteur est probablement la profondeur du graphe d'imports plutôt que le nombre brut de fichiers.

## Fix proposé

1. Profiler comply sur remix et nextjs pour identifier le bottleneck (import resolution ? graphe cyclique ?)
2. Ajouter un timeout interne par fichier ou par phase
3. Émettre un résultat partiel si le timeout est atteint plutôt que 0 output
4. Investiguer si c'est le debug build (`cargo run`) qui cause la lenteur — comparer avec release

## Fichiers concernés

- `src/engine/mod.rs` — timeout et résultats partiels
- `src/files.rs` — résolution d'imports, potentiel graphe cyclique
