# shadcn-* rules fire sans shadcn/radix installé

**Règles** : `shadcn-no-raw-tailwind-colors`, `shadcn-no-raw-tailwind-sizes`, et autres
**Impact** : ~123 faux positifs sur formik
**Priorité** : haute
**Lié à** : ISS-107 (même pattern — règles framework-spécifiques sans détection framework)

## Problème

Les règles préfixées `shadcn-*` recommandent d'utiliser les semantic tokens de shadcn/ui (`bg-primary`, `text-muted-foreground`) au lieu des classes Tailwind brutes. Mais formik n'utilise ni shadcn ni Radix UI — il utilise Tailwind directement.

```tsx
// website/src/components/Nav.tsx
<div className="bg-gray-800">  // ← flaggé "switch to shadcn semantic tokens"
// Formik n'a pas de shadcn tokens, ce conseil est impossible à suivre
```

## Fix proposé

Les règles `shadcn-*` ne devraient fire QUE quand shadcn/ui est détecté :
1. Présence de `components.json` (fichier config shadcn) à la racine
2. Ou dépendance `@radix-ui/*` dans package.json
3. Ou présence du dossier `components/ui/` avec le pattern shadcn

Sans cette détection, skip toutes les règles shadcn-*.

## Fichiers concernés

- `src/frameworks/mod.rs` — détection shadcn comme framework/library
- `src/rules/shadcn_*/` — gater sur la détection
