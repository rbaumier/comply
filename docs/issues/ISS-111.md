# filename-naming-convention FP sur conventions de routing Next.js

**Règle** : `filename-naming-convention`
**Impact** : ~68 faux positifs sur formik (dont 46 PascalCase .tsx, 5 pages/ conventions)
**Priorité** : moyenne
**Lié à** : ISS-096 (SvelteKit), ISS-080 (React PascalCase)

## Problème

ISS-096 couvre les conventions SvelteKit (`+page.svelte`), ISS-080 couvre les composants React PascalCase. Mais Next.js a ses propres conventions de nommage que la règle ignore :

### Pages Router
```
pages/_app.tsx          → flaggé (préfixe _)
pages/_document.tsx     → flaggé (préfixe _)
pages/[slug].tsx        → flaggé (crochets)
pages/[...params].tsx   → flaggé (crochets + spread)
pages/404.tsx           → ok (mais devrait être exclu comme convention)
```

### App Router
```
app/page.tsx           → ok
app/layout.tsx         → ok
app/loading.tsx        → ok
app/[slug]/page.tsx    → flaggé (crochets dans le dossier)
```

## Fix proposé

Élargir ISS-096 pour reconnaître les conventions de TOUS les frameworks de routing :
1. Next.js : `_app`, `_document`, `[param]`, `[...catch]`, `[[...optional]]`
2. Nuxt : `[id].vue`, `[...slug].vue`
3. Remix : `$param.tsx`, `_layout.tsx`
4. Astro : `[...slug].astro`

Ou accepter que les fichiers dans un dossier `pages/` ou `app/` suivent les conventions de leur framework.

## Fichiers concernés

- `src/rules/filename_naming_convention/` — whitelist des patterns de routing
