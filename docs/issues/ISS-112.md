# no-duplicate-string FP dans les tests Rust inline (#[cfg(test)])

**Règle** : `no-duplicate-string`
**Impact** : ~869 faux positifs sur starship
**Priorité** : haute

## Problème

En Rust, les tests sont inline dans le même fichier que le code, dans un module `#[cfg(test)]`. Les strings répétées dans les tests sont normales — ce sont des fixtures, des noms de modules, des configs de test :

```rust
// src/modules/git_status.rs — ligne 935, dans #[cfg(test)] mod tests
// "git_status" apparaît 49 fois dans les tests du module git_status
let actual = ModuleRenderer::new("git_status").config(config).collect();
```

La règle flagge `"git_status"` 49 fois, mais extraire cette string dans une constante rendrait les tests moins lisibles sans aucun bénéfice.

Sur starship, 869 des 895 violations `no-duplicate-string` sont dans des sections `#[cfg(test)]`.

## Fix proposé

1. Ne pas compter les strings dupliquées dans les blocs `#[cfg(test)]` pour les fichiers Rust
2. Plus largement, ne pas compter les strings dans les fichiers de test (`.test.ts`, `.spec.ts`, `__tests__/`) — les tests ont naturellement des strings répétées

## Fichiers concernés

- `src/rules/no_duplicate_string/` — exclure les sections de test
