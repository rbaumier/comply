# rust-impl-debug-on-public-types FP sur les crates binaires

**Règle** : `rust-impl-debug-on-public-types`
**Impact** : ~112 faux positifs sur starship
**Priorité** : moyenne

## Problème

La règle flagge les types `pub` sans `Debug` impl en disant "consumers can't log it". Mais dans un crate binaire (`main.rs`), `pub` signifie "visible aux autres modules du même crate", pas "partie de l'API publique".

```rust
// src/configs/dart.rs — dans un binary crate
#[derive(Clone, Deserialize, Serialize)]
pub struct DartConfig<'a> { ... }
// → flaggé "has no Debug impl" — mais personne ne consomme ce type en externe
```

Starship est un CLI (`main.rs`). Ses 112 config structs sont `pub` pour l'accès inter-modules, mais n'ont aucun consommateur externe.

## Fix proposé

1. Détecter si le crate est un binary (présence de `main.rs` ou `[[bin]]` dans Cargo.toml)
2. Si binary crate : ne pas flagguer, ou réduire la sévérité à info
3. Ne flagguer `pub` types sans Debug que dans les library crates (`lib.rs`)

## Fichiers concernés

- `src/rules/rust_impl_debug_on_public_types/` — détecter binary vs library crate
