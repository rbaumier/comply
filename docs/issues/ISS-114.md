# rust-partial-eq-without-eq FP sur les structs de test

**Règle** : `rust-partial-eq-without-eq`
**Impact** : ~181 faux positifs sur clap (tests/)
**Priorité** : moyenne

## Problème

Les structs de test dérivent `PartialEq` pour pouvoir utiliser `assert_eq!`, mais n'ont pas besoin de `Eq` car elles ne sont jamais utilisées dans un `HashSet` ou `BTreeSet` :

```rust
// tests/builder/action.rs
#[derive(Clone, Debug, PartialEq)]
struct Foo { bar: u32 }
// → flaggé "implements PartialEq but not Eq"
```

Sur clap, 181 des 187 violations sont dans `tests/`. Ce sont des structs jetables pour les assertions.

## Fix proposé

1. Ne pas flagguer dans les fichiers/modules de test
2. Ou réduire la sévérité à info dans les tests

## Fichiers concernés

- `src/rules/rust_partial_eq_without_eq/` — skip tests
