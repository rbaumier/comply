# filename-naming-convention FP sur les exemples binaires Rust (kebab-case)

**Règle** : `filename-naming-convention`
**Impact** : ~55 faux positifs sur clap
**Priorité** : basse

## Problème

En Rust, le nom de fichier d'un exemple dans `examples/` devient le nom du binaire. Les conventions CLI utilisent kebab-case pour les noms de binaires. Le filename-naming-convention exige snake_case pour les fichiers Rust, ce qui entre en conflit :

```
examples/cargo-example-derive.rs   → binaire: cargo-example-derive ✓
examples/cargo_example_derive.rs   → binaire: cargo_example_derive ✗ (pas convention CLI)
```

Ce pattern est standard dans l'écosystème Cargo : le `Cargo.toml` de clap définit `[[example]]` avec des noms kebab-case.

## Fix proposé

1. Accepter kebab-case pour les fichiers Rust dans `examples/` et `bin/`
2. Ou ne pas flagguer les fichiers qui correspondent à des entrées `[[example]]` ou `[[bin]]` dans Cargo.toml

## Fichiers concernés

- `src/rules/filename_naming_convention/` — whitelist kebab-case pour examples/ et bin/
