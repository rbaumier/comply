# id-length trop strict sur les paramètres de closure Rust

**Règle** : `id-length`
**Impact** : ~1063 faux positifs sur clap, ~48 sur starship
**Priorité** : haute

## Problème

La règle exige des identifiants de longueur ≥ 2 caractères. En Rust, les paramètres de closure à un caractère sont idiomatiques et omniprésents :

```rust
// Paramètres de closure — convention Rust standard
args.iter().filter(|a| a.is_valid())
values.map(|v| v.to_string())
result.unwrap_or_else(|e| panic!("{e}"))

// Iterator variables
for (k, v) in map.iter() { ... }

// Type parameters dans les impls
impl<T> Foo<T> { fn bar(&self, f: impl Fn(T)) { ... } }
```

Tous ces patterns sont recommandés par le Rust Style Guide et universels dans l'écosystème. 1063 violations sur clap seul montre que la règle est incompatible avec le code Rust idiomatique.

## Fix proposé

1. Exempter les paramètres de closure (`|x|`, `|e|`, `|f|`) de la vérification de longueur
2. Exempter les variables de loop/iterator (`for x in`, `for (k, v) in`)
3. Ou exempter les fichiers Rust entièrement (le compilateur Rust + clippy ont déjà leurs propres conventions)

## Fichiers concernés

- `src/rules/id_length/` — exemptions pour closures et iterators Rust
