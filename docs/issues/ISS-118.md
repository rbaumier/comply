# no-default-export FP sur fichiers config et routing Next.js/Remix

**Règle** : `no-default-export`
**Impact** : ~42 faux positifs sur create-t3-app
**Priorité** : haute

## Problème

La règle interdit les default exports. Mais plusieurs conventions de l'écosystème EXIGENT un default export :

### 1. Fichiers de config
```typescript
// drizzle.config.ts — drizzle-kit REQUIERT default export
export default defineConfig({ ... })

// eslint.config.js — flat config REQUIERT default export
export default tseslint.config({ ... })

// next.config.js, tailwind.config.ts, vite.config.ts, etc.
```

### 2. Routing Next.js (App Router)
```tsx
// app/page.tsx — Next.js REQUIERT default export pour le composant de page
export default function HomePage() { ... }

// app/layout.tsx, app/loading.tsx, app/error.tsx — idem
```

### 3. Routing Remix/Astro
```tsx
// routes/_index.tsx — Remix REQUIERT default export
export default function Index() { ... }
```

Sur create-t3-app, 42 des 52 violations sont sur ces patterns.

## Fix proposé

Exclure `no-default-export` pour :
1. `*.config.{js,ts,mjs}` et `.*.{js,ts,mjs}` (eslintrc, etc.)
2. Fichiers dans `app/` ou `pages/` (routing Next.js)
3. Fichiers dans `routes/` (Remix, SvelteKit)
4. Fichiers `page.tsx`, `layout.tsx`, `loading.tsx`, `error.tsx`, `route.ts`

## Fichiers concernés

- `src/rules/no_default_export/` — whitelist par nom de fichier et dossier
