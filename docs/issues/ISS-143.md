# number-literal-case applique les conventions JS au Rust

**Règle** : `number-literal-case`
**Impact** : 3976 rustls, 986 firecracker — lowercase hex digits en crypto/systems code
**Priorité** : critique

## Problème

La règle demande `0xFFU8` au lieu de `0xFFu8`. Mais en Rust, le suffixe de type est **toujours en minuscule** : `u8`, `i32`, `f64`, `usize`. `U8` n'existe pas dans le langage.

```rust
// Correct en Rust :
let buf = [0xFFu8; KsuidMs::PAYLOAD_BYTES];
// La règle veut :
let buf = [0xFFU8; ...];  // ← ne compile PAS en Rust
```

La règle applique la convention JavaScript (où les hex digits sont en majuscule et le suffixe n'existe pas) au Rust (où le suffixe de type est un token du langage, obligatoirement en minuscule).

## Fix proposé

Ne pas appliquer cette règle aux fichiers Rust, ou ne normaliser que les hex digits (a-f → A-F) sans toucher au suffixe de type.

## Fichiers concernés

- `src/rules/number_literal_case/` — exclure les suffixes Rust du renaming
