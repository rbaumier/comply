# top-level-function trop strict — arrow functions module-level légitimes

**Règle** : `top-level-function`
**Impact** : 1556 documenso, 1159 fp-ts — flag chaque arrow function exportée au module scope
**Priorité** : haute

## Problème

La règle veut que toute fonction top-level soit une `function declaration`. Mais de nombreux projets TS utilisent des arrow functions exportées par convention :

```typescript
// Flaggé — mais parfaitement lisible
export const createUser = async (data: UserInput) => { ... };
export const formatDate = (date: Date) => date.toISOString();

// La règle veut :
export async function createUser(data: UserInput) { ... }
```

Sur documenso (1556 hits) et fp-ts (1159 hits), c'est le style dominant du projet.

## Fix proposé

1. Ne pas flaguer les arrow functions assignées à une `const` exportée
2. Ou réduire la sévérité à `info` par défaut
