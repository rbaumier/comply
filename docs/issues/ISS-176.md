# ts-no-redeclare + no-function-overloads FP massifs sur les bibliothèques de types

**Règles** : `ts-no-redeclare`, `no-function-overloads`
**Impact** : 3666 + 1449 = 5115 FP sur fp-ts seul ; ts-pattern aussi touché
**Priorité** : haute

## Problème

fp-ts est une bibliothèque FP qui utilise massivement les surcharges de fonctions et les déclarations multiples pour le type inference :

```typescript
// Pattern standard fp-ts — chaque overload affine le type de retour
export function pipe<A>(a: A): A
export function pipe<A, B>(a: A, ab: (a: A) => B): B
export function pipe<A, B, C>(a: A, ab: (a: A) => B, bc: (b: B) => C): C
export function pipe(...args: any[]): any { ... }
```

`ts-no-redeclare` flagge chaque overload comme redéclaration.
`no-function-overloads` flagge l'usage de surcharges (veut une signature unique avec union types).

## Fix proposé

1. `ts-no-redeclare` : exclure les function overloads (déclarations suivies d'une implémentation)
2. `no-function-overloads` : réduire la sévérité ou ajouter un seuil minimum (>2 overloads = ok pour type inference)

## Fichiers concernés

- `src/rules/ts_no_redeclare/`
- `src/rules/no_function_overloads/`
