# rust-undocumented-unsafe trop strict — ne reconnaît pas les formats de SAFETY comment existants

**Règle** : `rust-undocumented-unsafe`
**Impact** : 479 tokio, 97 rayon — projets avec unsafe documenté mais format non reconnu
**Priorité** : moyenne

## Problème

La règle demande un commentaire `// SAFETY:` au-dessus de chaque bloc `unsafe`. Mais tokio et rayon documentent souvent avec des variantes :

```rust
// SAFETY: this is safe because...    ← reconnu ✓
// Safety: ...                         ← peut-être pas reconnu
// # Safety                            ← doc comment rustdoc, pas reconnu
/// # Safety
/// ...                                ← doc sur la fn, pas le bloc
```

479 hits sur tokio (qui documente scrupuleusement la safety) indique que le format reconnu est trop strict.

## Fix proposé

1. Reconnaître `// Safety:`, `// SAFETY:`, `# Safety` dans doc comments
2. Accepter un commentaire safety sur la fonction contenant le bloc unsafe (pas uniquement au-dessus du bloc)

## Fichiers concernés

- `src/rules/rust_undocumented_unsafe/`
