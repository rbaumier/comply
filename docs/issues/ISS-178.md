# i18n-no-hardcoded-string-in-jsx FP sur projets sans i18n ou avec strings non-traduisibles

**Règle** : `i18n-no-hardcoded-string-in-jsx`
**Impact** : 2049 documenso — flag chaque string dans du JSX
**Priorité** : haute

## Problème

La règle flagge toute string littérale dans du JSX comme devant utiliser i18n. Mais :

1. Certains projets n'utilisent pas (encore) d'i18n
2. Certaines strings ne sont pas traduisibles : noms de marque, codes, URLs, attributs techniques
3. Le volume (2049 hits) rend la règle inutilisable en pratique

```tsx
// Flaggé mais légitime
<Badge variant="outline">Pro</Badge>
<title>Documenso</title>
<Input placeholder="https://..." />
```

## Fix proposé

1. Ne pas activer la règle si aucune dépendance i18n n'est détectée (react-i18next, next-intl, etc.)
2. Exclure les strings courtes (<3 chars), URLs, noms de marque en PascalCase
3. Exclure les attributs non-visuels (placeholder, aria-label exceptions, data-*)
