# rust-no-lossy-as-cast trop bruyant sur code systèmes

**Règle** : `rust-no-lossy-as-cast`
**Impact** : 265 firecracker, 237 helix, 130 jiff, 33 rayon
**Priorité** : moyenne

## Problème

La règle flagge les casts `as` qui peuvent tronquer (ex: `u64 as u32`). Mais en code systèmes/bas-niveau, ces casts sont souvent intentionnels et documentés :

```rust
// Firecracker — registres hardware 32-bit
let reg_val = offset as u32;

// Helix — indices dans des buffers bornés
let col = pos.col as u16;  // col < terminal width, toujours < u16::MAX

// Jiff — conversions timestamp connues safe
let secs = nanos / 1_000_000_000 as i64;
```

## Fix proposé

1. Ne pas flaguer si un commentaire de justification est présent sur la même ligne
2. Exclure les casts vers des types plus grands (u32 → u64 = toujours safe)
3. Réduire la sévérité à `info` dans les fichiers non-API (modules internes)

## Fichiers concernés

- `src/rules/rust_no_lossy_as_cast/`
