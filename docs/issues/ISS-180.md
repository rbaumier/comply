# ts-no-shadow FP sur type parameters et closures imbriquées

**Règle** : `ts-no-shadow`
**Impact** : 2451 fp-ts, 163 ts-pattern
**Priorité** : moyenne

## Problème

La règle flagge les variables qui shadowed un nom du scope parent. Mais en TS type-heavy, les type parameters réutilisent souvent les mêmes noms (`A`, `B`, `E`) :

```typescript
// fp-ts — chaque fonction définit ses propres type params
const map = <A, B>(f: (a: A) => B) => <E>(fa: Either<E, A>): Either<E, B> => ...
// `A`, `B`, `E` réutilisés dans les closures imbriquées — shadow légitime
```

## Fix proposé

1. Ne pas flaguer le shadowing de type parameters génériques (`A`-`Z`, single-letter)
2. Exclure le shadowing dans les closures imbriquées quand le nom est un type parameter
