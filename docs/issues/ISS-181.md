# prefer-type-over-interface trop opinionated — style preference pas un bug

**Règle** : `prefer-type-over-interface`
**Impact** : 1588 tanstack-router, potentiellement massif sur tous les projets TS
**Priorité** : moyenne

## Problème

La règle force l'utilisation de `type` au lieu de `interface`. Mais les deux sont légitimes en TypeScript, et `interface` a des avantages spécifiques (declaration merging, extends, meilleur error messages).

```typescript
// Flaggé — mais parfaitement correct
interface UserProps {
  name: string;
  email: string;
}

// La règle veut :
type UserProps = {
  name: string;
  email: string;
};
```

De nombreux projets (tanstack, Angular, NestJS) utilisent `interface` par convention.

## Fix proposé

Réduire la sévérité à `info` ou rendre la règle opt-in au lieu de opt-out.
