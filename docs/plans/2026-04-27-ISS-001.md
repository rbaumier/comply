# ISS-001 Implementation Plan — Framework Entry Points for `unused-file` / `dead-export`

## Goal

Eliminate ~268 false positives (148 `unused-file` + 120 `dead-export`) on le dataset Amadeo en enseignant à comply les conventions TanStack Router (file-based routing), shadcn/ui (bibliothèque locale), les fichiers de config racine (dans `dead-export`), et les templates email.

---

## Step 1 — Extend `FrameworkDef` schema (additive, backwards compatible)

File: `src/frameworks/mod.rs`

Add three new optional fields:

```toml
[detection]
files = ["components.json"]   # file presence at project root

[entry_points]
file_suffixes = [".lazy.tsx", ".lazy.ts"]  # match file name endings
```

Only 2 new fields (the rest already exists in `FrameworkDef`):

1. Add `files: Vec<String>` to `Detection` struct (defaults to empty vec). This enables file-presence detection (e.g. `components.json`).
2. Add `file_suffixes: Vec<String>` to `EntryPoints` struct. This enables suffix matching (e.g. `*.lazy.tsx`).

Plumbing:
3. Update `detect_frameworks` signature: `pub fn detect_frameworks(pkg: &PackageJson, project_root: Option<&Path>) -> Vec<&'static FrameworkDef>`. A framework matches if any dependency matches OR any declared file exists at project root. File check uses `metadata().is_ok()` only.
4. Update `ProjectCtx::load` (`src/project/mod.rs:~384`) to pass `root.as_deref()`.
5. Add helper `pub fn framework_entry_file_suffixes(&self)` on `ProjectCtx`.
6. **Update 4 existing tests** in `src/frameworks/mod.rs` (~lines 110-136): `detects_nextjs`, `detects_jest_in_dev_deps`, `no_match_with_empty_pkg`, `multiple_frameworks_match` — all call `detect_frameworks(&pkg)` and need the new `None` second arg.

→ verify: existing tests pass with `None` as root, no behavioral change yet.

## Step 4 — Share `is_config_file` + skip framework entry dirs in `dead_export`

### 4a — Extract `is_config_file`

Move `is_config_file()` from `src/rules/unused_file/text.rs:122-134` into `FileCtx` (or a shared `src/rules/path_classification.rs`). Expose as `ctx.file.is_config_file()`.

Call sites to update:

2. `src/rules/unused_file/text.rs:78` — existing call
3. `src/rules/dead_export/text.rs` — new call (below)

### 4b — Add config-file skip in `dead_export`

In `src/rules/dead_export/text.rs`, after the `ctx.file.path_segments.in_test_dir` skip (line 32-34), add:

```rust
if ctx.file.is_config_file() {
    return Vec::new();
}
```

### 4c — Add `framework_entry_dirs` skip in `dead_export` (symmetric to Step 5)

This is critical: TanStack route files export `Route`, `loader`, etc. which are covered by `magic_exports`, but any custom helper export would still be flagged. Skip the entire file if it's inside a framework entry dir:

```rust
let path_str = ctx.file.canonical_path.to_string_lossy();
if ctx.project.framework_entry_dirs().any(|dir| path_str.contains(dir)) {
    return Vec::new();
}
```

Note: `dead_export::is_entry_point` currently takes `Option<&Path>` — for 4c, pass `&ctx.project` directly in the `check()` method body rather than extending `is_entry_point`'s signature.

→ verify: `vite.config.ts` no longer flagged; route files in `/routes/` no longer flagged.
